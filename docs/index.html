Phase 1 — Content Model: Scenes, Performers, Studios
Complete ✅
Redesign the core data model from Movie/TV to the adult content paradigm. Adult content is fundamentally different: it's organized by scenes (not movies), performers (not actors in the traditional sense), studios/channels (not networks), and series/collections (not TV seasons). The database schema, API entities, and TypeORM migrations must be rewritten.
Design new Entity schema: Scene, Performer, Studio, Channel, Collection, Release
Create TypeORM entities replacing Movie/TV with Scene/Collection models
Write database migration: preserve users/settings, transform media tables (SQLite + PostgreSQL)
Define content categories: Scene, Movie, Channel Release, JAV, Hentai, VR, Amateur, Gay, Trans, UHD
Implement Performer ↔ Scene many-to-many with roles (starring, featuring, directing)
Studio/Channel entity with logo, parent studio, network relationships
Series entity: scene grouping (e.g., "Tushy Raw V1–V99") with ordering — renamed from Collection to avoid confusion with UserCollection
scene_categories join table: normalized category support replacing comma-separated strings for efficient faceted filtering
Update API spec (voyeurr-api.yml) with new models and endpoints

Phase 2 — Whisparr Integration (The *arr Stack)
Complete ✅
Whisparr is the adult-content fork of Sonarr/Radarr in the *arr ecosystem. It handles monitoring, searching, grabbing, and renaming adult content. Voyeurr must talk to Whisparr the same way Seerr talks to Radarr/Sonarr — via the unified *arr API.

Eros branch research complete: Whisparr's Eros branch (1,881 commits ahead of v2) adds native Scene/Performer/Studio endpoints — GET /lookup/scene, POST /performer, POST /studio. stasharr (111★ reference userscript) confirms the *arr API pattern works identically with foreignId linking.
Study Whisparr API (v2 + Eros branch) + stasharr reference implementation
Create WhisparrAPI client class (mirrors RadarrAPI/SonarrAPI + Eros endpoints)
Implement quality profile sync, root folder discovery, tag management (via ServarrBase)
Build scene-to-Whisparr-movie mapping via foreignId (tpdb:/stash: prefix)
Create WhisparrScanner for availability sync (foreignId-based matching)
Implement request flow: Voyeurr request → Whisparr add + search → monitor status
Add Whisparr settings UI (server list, profiles, root folders, tags)
Handle multi-scene collections: batch-add to Whisparr with collection tags
Support 4K/VR quality profiles with separate Whisparr instances

Key adult trackers: AdultEmpire, Pornbay, Empornium, PornoLab, HappyFappy, xxxClub, YourExotic, PornLeech — plus general trackers with adult categories. Usenet: Slug, Dog, Geek, Finder (all have adult categories).
Research Jackett API (Torznab + Newznab) and Prowlarr API

Priority-based indexer selection (fastest first, fallback chain)

Content Categories & Metadata Model

Taxonomy

Adult content requires a fundamentally different taxonomy than mainstream media. Below is the complete category system Voyeurr will support, mapped to metadata providers and Whisparr.

🎬 Primary Content Types
  • Scene — Single scene release (most common). 20–60 min. Studio-produced or amateur.
  • Movie / Feature — Full-length adult film. 60–180 min. Narrative structure. DVD/VOD release.
  • Channel Release — Recurring series from a specific channel/studio (e.g., "Tushy Raw V47").
  • Collection / Series — Grouped releases. Ordered. E.g., all "Blacked Raw" episodes.
  • Compilation — Edited supercut. Multi-scene, multi-performer highlight reel.
🌍 Regional / Genre Categories
  • JAV — Japanese Adult Video. Censored/uncensored. Studio codes, actress IDs (DMM/FANZA).
  • Hentai — Animated adult. 2D/3D. Manga doujinshi, anime OVAs, visual novels.
  • Western — US/EU professional productions. Brazzers, Vixen, Blacked, etc.
  • Amateur / OnlyFans — Self-produced. Creator-direct. Clips and full scenes.
  • VR (Virtual Reality) — 180°/360° stereoscopic. Special playback requirements.
🎯 Target Audience Categories
  • Straight — Male/female, female/female content
  • Gay — Male/male content
  • Trans / TS — Transgender performer content
  • Queer / Bi — Mixed-gender, multi-partner scenes
🎥 Technical / Format Categories
  • 4K / UHD — 2160p resolution. Separate tracking from 1080p in Whisparr.
  • 60fps — High frame rate scenes. Whisparr custom format.
  • Uncensored — JAV specifically. Mosaic-removed or originally uncensored.
  • Leaked — Unreleased/private content. Controversial — optional, off by default.
  • Remastered — Upscaled/restored older scenes (AI-enhanced).
